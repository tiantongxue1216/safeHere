## 🚀 部署所需步骤

### 第 1 步：准备工作（10 分钟）
1. 注册微信小程序 → 获取 AppID
2. 开通微信云开发 → 记录环境 ID
3. 注册阿里云 → 申请千问 API Key

### 第 2 步：修改配置（5 分钟）
1. `project.config.json` → 填写 AppID
2. `app.js` → 填写云环境 ID
3. `cloudfunctions/createPost/index.js` → 填写 API Key
4. `cloudfunctions/createComment/index.js` → 填写 API Key

### 第 3 步：部署云函数（10 分钟）
1. 为云函数安装 npm 依赖
2. 上传并部署 createPost
3. 上传并部署 createComment

### 第 4 步：创建数据库（5 分钟）
1. 创建 posts 集合
2. 创建 comments 集合
3. 创建 encourages 集合
4. 设置权限和索引

### 第 5 步：测试运行（10 分钟）
1. 编译预览
2. 测试发帖
3. 测试鼓励
4. 测试评论
5. 真机测试语音

**详细步骤请参考：DEPLOYMENT.md**

## 💡 使用说明

### 发贴流程
```
1. 点击"💌 写下你的心声"
2. 输入文字或按住说话录音
3. （可选）选择表情
4. 点击"发布心声"
5. AI 自动审核（1-2 秒）
6. ✅ 发布成功
```

### 鼓励流程
```
1. 进入任意帖子详情
2. 点击"一键鼓励"蓝色按钮
3. 💙 → 💗（心跳动画）
4. 鼓励数 +1
5. 再次点击可取消
```

### 评论流程
```
1. 在底部输入框输入文字
2. 或点击麦克风录音
3. 点击"发送"
4. AI 自动审核
5. ✅ 评论成功
```

## 📝 重要提醒

### ⚠️ 必须修改的配置
1. **AppID** - 在 `project.config.json`
2. **云环境 ID** - 在 `app.js`
3. **千问 API Key** - 在两个云函数文件中

### 💾 数据库集合
需要手动创建以下集合：
- `posts` - 帖子数据
- `comments` - 评论数据
- `encourages` - 鼓励记录

### 🔐 安全提醒
- 妥善保管 API Key
- 不要上传敏感信息到公开仓库
- 定期备份数据库
- 监控云函数用量

- 数据库：2GB 存储

- **适合规模**: 日均 100-200 个帖子

### 阿里云千问 API
- 模型：qwen-turbo
- 价格：约 ¥0.008/千 tokens
- 单次审核：~100-200 tokens
- **估算**: ¥100 ≈ 6-12 万条内容审核
