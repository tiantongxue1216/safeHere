# 📊 创建数据库集合完整教程

## 🎯 目标
创建三个数据库集合来存储帖子、评论和鼓励数据

---

## 📝 步骤 1：打开云开发控制台

1. 在微信开发者工具顶部菜单栏
2. 点击 **"云开发"** 按钮（云朵图标 ☁️）
3. 等待控制台加载完成

---

## 📝 步骤 2：添加第一个集合 - posts（帖子）

### 2.1 创建集合
1. 在云开发控制台左侧菜单，点击 **"数据库"**
2. 点击 **"+ 添加集合"** 按钮
3. 输入集合名称：`posts`
4. 点击 **"确定"**

### 2.2 设置权限
1. 在 `posts` 集合页面，点击 **"权限设置"** 标签
2. 选择 **"自定义规则"**
3. 粘贴以下 JSON 代码：

```json
{
  "read": true,
  "write": "auth.openid == doc.userId"
}
```

**解释：**
- `"read": true` - 所有人都可以读取帖子（公开可见）
- `"write": "auth.openid == doc.userId"` - 只有发帖人自己能写入

4. 点击 **"保存"**

### 2.3 创建索引
1. 点击 **"索引"** 标签
2. 点击 **"添加索引"**
3. 添加以下两个索引：

**索引 1：**
- 字段名：`userId`
- 排序：升序

**索引 2：**
- 字段名：`createTime`
- 排序：降序

4. 点击 **"保存"**

## 💡 常见问题

### Q1: 提示"集合已存在"怎么办？
**A:** 说明集合已经创建过了，可以直接使用，无需重复创建。

### Q2: 权限设置错了能修改吗？
**A:** 可以随时修改！在集合页面点击"权限设置"重新配置即可。

### Q3: 索引忘记创建了有影响吗？
**A:** 初期不影响使用，但数据量大后会影响查询速度。建议补上。

### Q4: 能看到别人的数据吗？
**A:** 
- 帖子和评论是公开的（`read: true`）
- 但只有作者本人可以修改/删除自己的数据
- 这样设计是为了让社区内容可见，同时保护用户隐私
