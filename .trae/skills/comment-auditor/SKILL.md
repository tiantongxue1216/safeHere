# 评论审核器 (Comment Auditor)

## 功能描述

评论审核器是一个专门用于审核匿名倾诉社区中评论内容的技能，能够：

- **前置过滤**：快速过滤明显违规内容（毫秒级）
- **AI 精准审核**：使用阿里云千问大模型进行场景化审核
- **分级处理**：根据风险等级采取不同处理策略
- **保护用户**：防止二次伤害，营造安全的社区氛围

## 审核规则

### 第一层：前置合规过滤（毫秒级）

**硬拦截（直接拦截）**：
- 攻击性词汇：傻逼、SB、废物、垃圾、去死等
- 高危行为：自杀、自残、跳楼、割腕等
- 隐私信息：微信、QQ、电话、联系方式等
- 广告引流：HTTP链接、WWW链接等

**白名单（直接放行）**：
- 积极词汇：抱抱、加油、支持、理解、陪伴等
- 表情符号：🌸、💪、❤️、🤗、😊等

**快速规则**：
- 纯表情/纯标点内容
- 超短内容（1-2个字符）

**高风险（进入AI审核）**：
- 否定情绪：活该、矫情、玻璃心、想太多等
- 说教语气：你应该、你不应该、你要坚强等
- 对比言论：比你惨的多了、你这算什么等

### 第二层：AI 精准审核

使用阿里云千问大模型进行场景化审核，基于以下规则：

**必须拦截**：
1. 违法违规：涉政敏感、色情低俗、暴力血腥
2. 人身攻击：辱骂、诅咒、阴阳怪气、嘲讽
3. 二次伤害：否定情绪、指责受害者、洗白施害者
4. 高危引导：诱导自残、自杀、报复社会
5. 隐私引流：索要隐私、广告、引流外站
6. 歧视引战：性别/地域/疾病歧视，挑起对立

**必须放行**：
1. 共情安慰：表达共情、安慰、关心、支持、鼓励
2. 同频陪伴：分享相似经历、陪伴倾诉
3. 温和建议：不带指责、不说教，仅给温和理性建议
4. 中性内容：纯表情、纯标点、无不良导向的简短回应

**边界内容**：
- 轻微争议但无恶意：优先放行
- 无法明确判断：人工复核，不拦截
- 禁止将带有负面情绪的共情内容判定为拦截

## 集成方法

### 1. 安装依赖

```bash
# 云函数目录下执行
npm install axios
```

### 2. 配置 API Key

在使用前需要配置阿里云千问 API Key：

```javascript
// 在审核函数中设置
const QWEN_API_KEY = 'YOUR_QWEN_API_KEY' // 替换为你的 API Key
```

### 3. 调用示例

```javascript
// 导入审核模块
const { preFilterComment, auditCommentWithAI } = require('./comment-auditor');

// 前置过滤
const preResult = await preFilterComment(commentContent);
if (preResult.action === 'block') {
  return { success: false, message: preResult.reason };
}

// AI 审核（如果需要）
if (preResult.action === 'review') {
  const aiResult = await auditCommentWithAI(commentContent, postId);
  if (!aiResult.pass) {
    return { success: false, message: aiResult.reason };
  }
}

// 审核通过，继续处理
```

### 4. 自定义敏感词库

可以根据社区实际情况调整敏感词库：

```javascript
const SENSITIVE_WORDS = {
  hardBlock: ['傻逼', 'SB', '废物', '垃圾', '去死'],
  highRisk: ['活该', '矫情', '玻璃心', '想太多'],
  whitelist: ['抱抱', '加油', '支持', '理解', '陪伴']
};
```

## 审核流程

1. **接收评论**：获取评论内容和相关帖子信息
2. **前置过滤**：快速检查敏感词和规则
3. **AI 审核**：对高风险内容进行场景化分析
4. **结果处理**：根据审核结果决定是否放行
5. **记录日志**：保存审核记录便于后续分析

## 性能优化

- **缓存机制**：对常见评论模式进行缓存
- **批量处理**：支持批量审核提高效率
- **降级策略**：AI 服务不可用时的兜底方案
- **异步处理**：非关键路径采用异步审核

## 最佳实践

1. **定期更新**：根据社区反馈定期更新敏感词库
2. **人工复核**：对高风险但不确定的内容进行人工审核
3. **用户反馈**：建立用户举报机制，及时处理违规内容
4. **数据分析**：分析审核数据，优化审核规则

## 注意事项

- **API 费用**：千问 API 会产生调用费用，请注意控制用量
- **审核延迟**：AI 审核可能会有一定延迟，建议在前端添加加载提示
- **误判处理**：建立误判反馈机制，持续优化审核准确率
- **隐私保护**：审核过程中注意保护用户隐私，避免敏感信息泄露

## 输出格式

### 前置过滤结果

```javascript
{
  action: 'block' | 'pass' | 'review',
  reason: string, // 仅当 action 为 'block' 时
  rule: string // 命中的规则
}
```

### AI 审核结果

```javascript
{
  pass: boolean, // true 表示通过，false 表示拦截
  reason: string // 仅当 pass 为 false 时
}
```
