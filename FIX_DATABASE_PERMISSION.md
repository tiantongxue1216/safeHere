# 🔧 修复数据库权限错误：Permission denied

## ❌ 错误信息
```
errCode: -502003 database permission denied
errMsg: Permission denied
```

**错误原因**：数据库集合的权限设置不正确，导致云函数无法写入数据。

---

## ✅ 解决方案：设置数据库自定义权限

###  需要设置的集合

你需要为以下三个集合设置权限：
1. `posts` - 帖子集合
2. `comments` - 评论集合
3. `encourages` - 鼓励集合

---

## 🎯 详细操作步骤

### 步骤 1：打开云开发控制台

1. 在微信开发者工具顶部，点击 **"云开发"** 按钮 ☁️
2. 等待控制台加载完成

### 步骤 2：进入数据库页面

1. 在左侧菜单，点击 **"数据库"** 📊
2. 你会看到三个集合：`posts`、`comments`、`encourages`

---

### 步骤 3：设置 posts 集合权限

#### 3.1 打开权限设置
1. **点击** `posts` 集合名称
2. 在页面顶部，点击 **"权限设置"** 标签

#### 3.2 选择自定义规则
1. 在权限设置页面，找到 **"自定义规则"**
2. 点击 **"编辑"** 或 **"修改"**

#### 3.3 粘贴权限 JSON

复制以下 JSON 代码并粘贴：

```json
{
  "read": true,
  "write": "auth.openid == doc.userId"
}
```

#### 3.4 保存
点击 **"保存"** 按钮

---

### 步骤 4：设置 comments 集合权限

#### 4.1 打开权限设置
1. 返回数据库列表
2. **点击** `comments` 集合

#### 4.2 选择自定义规则
1. 点击 **"权限设置"** 标签
2. 点击 **"编辑"**

#### 4.3 粘贴权限 JSON

```json
  "read": true,

}
```

#### 4.4 保存
点击 **"保存"**

---

### 步骤 5：设置 encourages 集合权限 ⭐ 重点

2. **点击** `encourages` 集合

#### 5.2 选择自定义规则

## 💡 权限说明

### 为什么这样设置？

```json
{
  "read": true,                    // 公开可读
  "write": "auth.openid == doc.userId"  // 仅创建者可写
}
```

**这样设计的好处：**

| 权限 | 说明 | 好处 |
|------|------|------|
| `"read": true` | 所有人都可以读取 | 社区内容公开透明，所有人都能看到帖子和评论 |
| `"write": "auth.openid == doc.userId"` | 只有创建者可以写入/修改 | 保护用户隐私，防止他人篡改数据 |

### 具体场景举例：

#### ✅ 正常场景：
- 用户 A 发帖 → 写入 posts 集合 → **成功**（userId 匹配）
- 用户 A 给自己点赞 → 写入 encourages 集合 → **成功**（userId 匹配）
- 用户 B 评论用户 A 的帖子 → 写入 comments 集合 → **成功**（userId 匹配）
- 用户 C 浏览所有帖子 → 读取 posts 集合 → **成功**（read: true）

#### ❌ 被阻止的场景：
- 用户 B 尝试修改用户 A 的帖子 → **失败**（userId 不匹配）
- 用户 C 尝试删除用户 A 的评论 → **失败**（userId 不匹配）
- 用户 B 尝试给用户 A 的帖子点赞 → **成功**（因为鼓励记录的 userId 是用户 B 的）

## ⚠️ 常见问题

### Q1: 没有看到"权限设置"标签？
**A:** 确保你点击了具体的集合名称，不是在集合列表页面。

### Q2: 提示"权限不足"无法设置？
**A:** 
1. 检查你是否是云环境的管理员
2. 如果不是，联系环境创建者设置
3. 或者在云开发控制台 → 设置 → 成员管理中添加自己

### Q3: 设置后仍然报错？
**A:**
1. 等待 1-2 分钟让权限生效
2. 重新编译小程序
3. 清除缓存后重试
4. 检查 JSON 格式是否正确（不要有多余的空格或换行）

### Q4: JSON 格式错误？
**A:** 确保：
- 使用英文双引号 `"` 不是中文引号
- 逗号 `,` 后面没有多余的空格
- 整个 JSON 没有语法错误

## 🔒 安全提醒

### 不要设置以下权限：

❌ **完全公开**（危险！）：
```json
{
  "read": true,
  "write": true  // ❌ 任何人都可以修改任何数据
}
```

❌ **完全私有**（无法使用）：
```json
{
  "read": "auth.openid == doc.userId",  // ❌ 只有自己能看
  "write": "auth.openid == doc.userId"
}
```

✅ **正确的权限**：
```json
{
  "read": true,                    // ✅ 公开可读
  "write": "auth.openid == doc.userId"  // ✅ 仅创建者可写
}
```
